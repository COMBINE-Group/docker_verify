Common Input Parameters of the Analysis

This section groups all parameters in common among Model Verification Tools

  • Skip Rows: This value represents the number of rows that must be removed from the input files before proceeding with the analysis, and it is useful to remove unneeded values such columns headers or other information;
  • Column to Analyze: This parameter allows to select the column on which to perform the analysis
  • Separator character: it represents the field separator that is used to separate the columns in the uploaded files. The model Verification Tools support four separator (space, tab, comma and semicolon)
  • Seed: this parameter sets the random-seed for the generation of the sample-set

WARNING: The values inside the files must use the Decimal point separator. Otherwise, the analysis can not be performed.

WARNING: The values of the first column of all input files, represent the time.

Smooothness Analysis

The Smoothness Analysis can be used to evaluate the singularities, discontinuities and buckling errors inside your model.

Input Parameters

  • Window size: In order to detect possible local roughness in the model, our smoothness analysis perform a coefficient of variation using the k nearest neighbours in the windows. This parameter allow to choose the window size.
  • File to analyze: is a ASCII or CSV file that defines the model outputs in which the first column usually represents the time (expressed as a physical time or time-step number), and the other columns represent the outputs to be analyzed.

Time Step Convergence Analysis

This analysis allows to determine if there is a convergence in the behavior of the model outputs as the time-step length becomes narrower.

Input Parameters

  • File to analyze: The user can upload two or more ASCII or CSV files that defines the model outputs in which the first column represents the time, and the other columns represent the outputs to be analyzed. At the moment, Time Step Convergence Analysis consider the peak-value and the relative time-to-peak-value.

Uniqueness Analysis

The aims of this analysis is to check if repeated executions on the same input parameter set lead to identical output.

Input Parameters

  • File to analyze: The user can upload one or more files in ASCII or CSV format.

Sobol Analysis

Sobol Sensitivity Analysis methodology, is a technique to perform a variance based sensitivity analysis of model output.

Input Parameters for SOBOL sample generation

  • Number of combinations:The user use this parameter to set the number of samples to generate
  • Input parameter file: is a CSV file that defines the parameters of the model and its range values. The file must have a header and this structure:
    • param_name: is the name of the parameters
    • first_value: see the below table
    • second_value: see the below table
    • distribution: it represent the distribution for each parameter. You can choose among: "unif", "triang", "norm" and "lognorm"

WARNING: The columns first_value and second_value take on different meanings according to the chosen distribution. the following table shows the corresponding values according to the chosen distribution.

Distribution First value Second value
unif Minimum Maximum
triang Jacob Thornton
norm Mean Standard Deviation
lognorm Mean Standard Deviation

WARNING: The tools calculate the second-order sensitivities, so the output matrix has N * (2D + 2) rows. Where D is the number of parameters and N is The number of samples to generate. Further information, read the official documentation

Input Parameters for SOBOL analysis

  • Number of combinations: is the same of the sample generation
  • Input parameter file: is the same of the sample generation
  • Output files from the model: Here, the users upload the model outputs files, in ASCII or CSV format, without Header. In order to performs correctly the analysis it is important to rename the files correctly.
    The name of the output model files would be, ("0_1.csv", "0_2.csv",..., "o_n.csv"). For example, the "0_1.csv" file represent the output of the model that uses the first row values obtained from the Sample Generation output matrix.
    The "0_2.csv" file represent the output of the model that uses the second row values obtained from the Sample Generation output matrix and so on

Latin Hypercube Sampling Analysis and Partial Rank Correlation Coefficients Analysis

Input Parameters for Latin Hypercupe Sampling

Latin hypercube sampling is a statistical method for generating a near-random sample of parameter values from a multidimensional distribution.

  • Number of samples: Defines the number of samples
  • Iterations: Defines the number of iterations for optimizing LHS
  • Input parameter file: is a CSV file that defines the parameters of the model and its minimum and maximum values. The file must have a header and this structure:
    • param_name: is the name of the parameter
    • min: is the min value of the parameter
    • max: is the max value of the parameter

Input Parameters for Partial Rank Correlation Coefficients (Over time)

Partial Rank Correlation Coefficients is a Sensitivity Analysis test, it's a sampling-based method

  • Time points interval:specific the number of time points in which perform the analysis
  • Threshold p-value:this value is used as a threshold, below which the plots are not shown
  • File to analyze:The user can upload two or more ASCII or CSV files that defines the model outputs in which the first column represents the time, and the other columns represent the outputs to be analyzed.
  • LHS Matrix: it is a CSV output file from Latin Hypercube Sampling analysis

Input Parameters for Partial Rank Correlation Coefficients (Specific Time Step)

  • Time step:specific the time step in which perform the analysis
  • Threshold p-value:this value is used as a threshold, below which the plots are not shown
  • File to analyze:The user can upload two or more ASCII or CSV files that defines the model outputs in which the first column represents the time, and the other columns represent the outputs to be analyzed.
  • LHS Matrix: it is a CSV output file from Latin Hypercube Sampling analysis
